 Context

     The kg-rag stack (E:\Projects\Python\Multi-Source-Knowledge-Base-Graph) currently has no authentication — all /v1/* API routes and the frontend are
     open. CloudFoundry UAA (E:\Projects\Java\uaa) is a full OAuth2/OIDC server that will run standalone (./gradlew run from uaa/uaa, listening on
     http://localhost:8080/uaa, default HSQLDB). We'll wire it in as the identity provider:

     - FastAPI validates UAA-issued JWT access tokens (JWKS-based) on all /v1/* routes.
     - The frontend/ React app performs an OAuth2 Authorization Code + PKCE login against UAA and attaches the resulting access token to API calls.
     - UAA needs a new OAuth client registered for the SPA's redirect URI + CORS allowance.

     1. Register a UAA OAuth client (one-time setup, documented for the user)

     UAA's default app client supports authorization_code but its redirect-uri is http://localhost:8080/**,http://localhost:7000/**. We'll register a
     dedicated public SPA client instead.

     Add E:\Projects\Python\Multi-Source-Knowledge-Base-Graph\scripts\register-uaa-client.sh (bash, runnable via Git Bash) that:
     1. Gets an admin token: POST {UAA}/oauth/token with client_id=admin&client_secret=adminsecret&grant_type=client_credentials.
     2. Registers client kg-rag-frontend:
       - authorized-grant-types: authorization_code,refresh_token
       - redirect-uri: http://localhost:3001/callback
       - scope: openid,profile,email
       - autoapprove: [openid]
       - no client secret (public client, PKCE)
     3. Prints a reminder to add CORS for the SPA origin to uaa.yml (cors.xhr.allowed-origins / allowed-origin-patterns include ^http://localhost:3001$) and
     restart UAA — needed because the browser calls /oauth/token directly cross-origin during PKCE code exchange.

     Document these steps + default test user (marissa/koala) in the project README under a new "Authentication (UAA)" section.

     2. Backend: validate JWTs from UAA on /v1/* routes

     pyproject.toml — add to [tool.poetry.dependencies]:
     pyjwt = {extras = ["crypto"], version = "^2.9.0"}
     (requests already available for JWKS fetch.)

     libs/common/kg_rag_common/settings.py — add fields:
     uaa_issuer_url: str = Field(default="http://localhost:8080/uaa")
     uaa_jwks_url: Optional[str] = Field(default=None)  # defaults to f"{issuer}/token_keys"
     uaa_client_id: str = Field(default="kg-rag-frontend")

     New file apps/api/kg_rag_api/auth.py:
     - PyJWKClient (lru_cache'd) pointed at {uaa_issuer_url}/token_keys.
     - verify_token(credentials: HTTPAuthorizationCredentials = Depends(HTTPBearer())) dependency:
       - fetch signing key via JWKS client, jwt.decode(token, key, algorithms=["RS256"], issuer=uaa_issuer_url, options={"verify_aud": False})
       - raise HTTPException(401) on InvalidTokenError
       - return decoded claims dict (so routes can access sub, user_name, scope later if needed)

     apps/api/kg_rag_api/main.py — apply auth at router-inclusion time (no edits to individual route files needed):
     from apps.api.kg_rag_api.auth import verify_token
     ...
     app.include_router(docs_router, dependencies=[Depends(verify_token)])
     app.include_router(search_router, dependencies=[Depends(verify_token)])
     app.include_router(generate_router, dependencies=[Depends(verify_token)])
     app.include_router(roadmap_router, dependencies=[Depends(verify_token)])
     app.include_router(answer_router, dependencies=[Depends(verify_token)])
     app.include_router(agent_router, dependencies=[Depends(verify_token)])
     app.include_router(admin_api_router, dependencies=[Depends(verify_token)])
     health_router and ui_router remain unauthenticated (ops/legacy UI).

     infra/docker-compose.yml — add to api service environment:
     KG_UAA_ISSUER_URL: http://host.docker.internal:8080/uaa
     KG_UAA_CLIENT_ID: kg-rag-frontend
     (UAA runs on the host via gradlew, so the containerized API reaches it through host.docker.internal.)

     3. Frontend: Authorization Code + PKCE login

     frontend/package.json — add oidc-client-ts and react-oidc-context.

     frontend/.env.example — add:
     VITE_UAA_AUTHORITY=http://localhost:8080/uaa/oauth/token
     VITE_UAA_CLIENT_ID=kg-rag-frontend
     VITE_UAA_REDIRECT_URI=http://localhost:3001/callback
     (authority must point at the OIDC issuer that serves /.well-known/openid-configuration; for UAA this is {base}/oauth/token.)

     frontend/src/main.tsx — wrap <App /> in <AuthProvider> from react-oidc-context, configured from the env vars above with scope: "openid", redirect_uri,
     automatic PKCE.

     frontend/src/App.tsx —
     - Add <Route path="callback" element={<CallbackPage />} /> (new small component under src/pages/ that shows a spinner while react-oidc-context
     processes the redirect, then navigates to /).
     - Wrap the existing protected routes (documents, search, qa, roadmap, admin) in a new ProtectedRoute component (src/components/ProtectedRoute.tsx) that
     uses useAuth() — if not authenticated, calls auth.signinRedirect(); shows a loading state while isLoading.

     frontend/src/services/api.ts — add a module-level token getter set by an AuthSync effect (small component rendered inside AuthProvider, calls useAuth()
     and stores user?.access_token in a module variable via an exported setter). Update the request interceptor:
     config.headers.Authorization = getAccessToken() ? `Bearer ${getAccessToken()}` : undefined;

     frontend/src/components/layout/Header.tsx — add a user/login area using useAuth(): shows username + "Sign out" (auth.signoutRedirect() / removeUser())
     when authenticated, or "Sign in" (auth.signinRedirect()) when not.
